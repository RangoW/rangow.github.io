codesign

Mac OS App CodeSign

codesign tool

  • codesign --remove-signature <code-path>

    删除内容如图所示：

  • 对删除后的app，使用 codesign -s <identity> <code-path> 重签

    与原生签名app比较，如图所示：

  • 验证签名，codesign -vv -d --deep xxx.app [--verbose=20] 验证结果请自行实验

  • 使用 ‘-r’ 选项指定签名要求，codesign -s <identity> -r="host => anchor apple and identifier com.apple.perl" <code-path>

info.plist

表明程序运行的指定要求，至少包括以下两项

  • CFBundleIdentifier
    –ident, com.microfost.VSCode 反向DNS表示法，以top-level域名开始，后跟公司名称，最后是产品名称
  • CFBundleName

    应用程序的简短名称，不超过16个字符

签名内容

  • 嵌套代码，递归性签名应用程序依赖或绑定的所有helper、tools、libraries、framework and other components

  • Mach-O 可执行程序，将单个签名应用于可执行程序的每个体系结构。但通常只验证最终用户本机系统架构对应的签名

  • 资源文件，应用程序中非显示代码文件都是资源文件。所有资源文件的签名统一放在一个plist文件中，Contents/_CodeSignature/CodeResources

Entitlements

  • 授权文件，授予app特定权限，从而启用app特殊功能，比如启用icloud、推送通知、Apple Pay、App SandBox
